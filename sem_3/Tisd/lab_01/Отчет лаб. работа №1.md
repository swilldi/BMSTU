# Описание условий задачи
Реализовать операцию деления целого числа длиной до 40 десятичных цифр на действительное число в форме ±m.n Е K, где суммарная длина мантиссы (m+n) - до 40 значащих цифр, а величина порядка K - до 5 цифр. Результат выдать в форме ±0.m1 Е ±K1, где m1 - до 40 значащих цифр, а K1 - до 5 цифр.
# Техническое задание
### Исходные данные
**Действительное число**: строка состоящая из символов цифр и символов ".-+". Удовлетворяет формату \[+-]n.m e \[+-]k, где (порядок n + порядок m) $\le$ 40, порядок k $\le$ 5. Если знак перед числом не указан, что считается, что число положительное. Незначащие нули до целой части и после дробной не учитываются. Строка может содержать пробельные символы до или после полного числа.
Доступные форматы ввода: 123; 123.456; 123.456e7; +123.456e7; -123.456e7; 0.123e456; .123e456; 00234.324000e34; и т.п.
**Целое число**: строка состоящая из символов цифр и символов "-+". Удовлетворяет формату $\pm$ n, где порядок n не более 40. Если знак перед числом не указан, что считается, что число положительное. Незначащие нули до целой части. Строка может содержать пробельные символы до или после полного числа.
Доступные форматы ввода: 123; -123; +123; 123; 00123 и т.п.
### Выходные данные
Вывод в консоль действительного числа в формате 0.MeK, порядок числа M $\le$ 40, порядок K$\le$ 5
### Описание реализуемой задачи
Необходимо реализовать деление целого "длинного" числа на "длинное" действительное число. Для выполнения операции используется деление "в столбик". Для обработки входных данных используется 2 структуры: 
- Структуру мантисы, которая содержит:
	- Цифры входящие в манису числа, записанные в обратном порядке;
	- Длину мантиссы. 
- Структуру "длинного" числа, которая содержит
	- Структуру мантиссы;
	- Знак числа;
	- Значение экспоненты.
### Способы обращения к программе
Запуск через терминал с помощью команды "./app.exe". После этого необходимо ввести два числа: делимое и делитель.

# Описание структуры данных
```C
#define DIGITS_COUNT_MAX 40
#define EXPONENTA_COUNT_MAX 5
typedef struct
{
    uint8_t digits[DIGITS_COUNT_MAX + 1];
    size_t len;
} ulong_num_t;

typedef struct
{
    ulong_num_t mantissa;
    int exponent;
    bool sign;
} large_num_t;
```

### Поля структуры `ulong_num_t`
 - **digits** – массив цифр числа, записанные в обратном порядке
 - **len** – количество цифр в числе
### Поля структуры `large_num_t`
- **mantissa** – мантисса числа
- **exponent** – значение экспоненты нормализованного числа
- **sign** – знак числа
# Описание алгоритма
### Деление "длинных" чисел
1) Проверка, что делитель не является нулем. Иначе вернуть не нулевой код
2) Проверка, что делимое не является нулем. Иначе заполнить результат, как 0 и вернуть нулевой код
3) Присвоить начальные значения длины мантиссы, знака и экспоненты для результата
4) Получить неполный делитель
5) Найти максимального значения  t, такого что произведения делителя на t меньше неполного делимого
6) Из неполного делителя вычесть найденое произведение в пункте 5
7) В результат записать t
8) Добавить следующее число к неполному делимому
9) Повторять с пункта 5 пока (остаток не будет равен нулю и не закончатся цифры в делителе) или (длина мантиссы результата не превышает максимального значения)
10) Если мантисса длиннее максимальной длины, то выполнить округление
11) Форматировать результат
### Вычитание "длинных" чисел
1) Результату присвоить длину уменьшаемого числа
2) Установить флаг "занятие"
3) Для каждой цифры выполнить
	1) Получить результат вычитания цифр с учетом флага
	2) Если значение отрицательное, то прибавить в нему 10, и поднять флаг "занятие". Иначе опустить флаг "занятие"
	3) Сохранить значение вычитания в результат
4) Удалить незначащие нули и уменьшить длину результата
### Умножение длинного числа на цифру
1) Результату задать длину множителя
2) Для каждой цифры выполнить
	1) Перемножить две цифры
	2) Если результат больше 10, то записать остаток от деления на 10, целую часть от деления на 10 добавить к следующему умножению
# Необходимые функции
- `void ulong_num_multiply_digit(ulong_num_t *multiplicand, **int** multiplier, ulong_num_t *res);` – умножение мантиссы, на цифру
	- multiplicand – Число, которое умножаем
	- multiplier – Цифра на которую умножаем
	- res – Результат произведения
- `void ulong_num_subtract(ulong_num_t *minuend, ulong_num_t *subtrahend, ulong_num_t *res);` – вычитание меньшей мантиссы из большей
	- minuend – Уменьшаемое
	- subtrahend – Вычитаемое
	- res – Разность
- `int large_num_divide(large_num_t *dividend, large_num_t *divider, large_num_t *res);` – деление длинных чисел
	- dividend – Делимое
	- divider – Делитель
	-  res – Частное
- `void large_num_round(large_num_t *num);` – округление числа
	- num – Число, которое будет округлено
- `void print_large_num(**const** large_num_t *num);` – выводит структуру в виде 0.NeK, N – мантисса числа; K – значение экспонеты
	- num – Число значение, которого будет выведено
- `int input_large_num(large_num_t *num, NUMBER_TYPE t);` – Ввод строки, проверка данной строки и заполнение переменой num, если строка удовлетворяет переданому типу
	- num – Структура в которую будет записано число
	- t – Тип числа (Int, Float)
- `int str_is_valid(char *str, NUMBER_TYPE type);` –  проверка, что строка удовлетворяет формату 
	- $\pm n.mE\pm k$, $(n + m) \le 40$, $|k| \le 99999$ для дробных чисел
	- $\pm nE \pm k$, $n \le 40$, $|k| \le 99999$ для целых чисел
	- str – Строка, которую проверяют
	- type – Тип числа (Int, Float)
- `void str_to_large_num(char *str, large_num_t *large_num);` – парсинг строки в структуру для хранения "длинных" чисел
	- str – Строка, из которой будут получены значения числа
	- large_num – Структура, в которую будут перенесены значения числа из строки

# Выводы
Для работы с числами, превышающими разрядную сетку компьютера, можно использовать алгоритмы, которые обрабатывают числа поразрядно. "Длинные" числа можно приводить к нормализованному виду и для хранения и обработки использовать структуры, которые содержат значения мантиссы, экспоненты и знака числа 
# Контрольные вопросы
### Каков возможный диапазон чисел, представляемых в ПК?
- Для 32-х разрядный компьютеров, диапазон от $-2^{31}$ до $2^{31} - 1$ для знаковых и от $0$ до $2^{32} - 1$ для беззнаковых целых чисел. И приблизительно от $\pm1.4*10^{-45}$ до $\pm3.4*10^{38}$ для вещественных
- Для 64-х разрядный компьютеров, диапазон от $-2^{63}$ до $2^{63} - 1$ для знаковых и от $0$ до $2^{64} - 1$ для беззнаковых целых чисел. И приблизительно от $\pm5.0*10^{-324}$ до $\pm1.7*10^{308}$ для вещественных
### Какова возможная точность представления чисел, чем она определяется?
Целые значения определяются точно. Точность вещественных зависит от количества бит выделенных под мантиссу. В 32-х битной системе для действительных чисел выделяется до 23 бит, а в 64-х битной до  52 бит.

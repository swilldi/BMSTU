# Описание условий задачи

Создать таблицу, содержащую не менее 40 записей с вариантной частью. Произвести поиск информации по вариантному полю. Упорядочить таблицу, по возрастанию цены автомобиля, используя: **а)** исходную таблицу; **б)** массив ключей, используя 2 разных алгоритма сортировки (простой, ускоренный). Оценить эффективность этих алгоритмов (по времени и по используемому объему памяти) при различной реализации программы, то есть, в случаях **а)** и **б).** Обосновать выбор алгоритмов сортировки.
# Техническое задание
### Исходные данные

txt файл, в котором содержатся данные о автомобилях в формате "бренд;номер\_страны;поддержка\_обслуживания;сервис;цена;цвет;состояние;гарантия;год\_выпуска;пробег;количество\_владельцев;количество\_ремонтов". Поле "поддержка\_обслуживания" указывается, если автомобиль является иномаркой. Поле "гарантия" указывается только при условии, что состояние автомобиля новое. Поля "год\_выпуска", "пробег", "количество\_владельцев", "количество\_ремонтов" указываются только, если состояние автомобиля не новое.
### Выходные данные
#### Неотсортированная таблица
Вывод таблицы, которая была считана из файла
#### Сортированная таблица ключей
Вывод сформированной таблицы ключей по исходной таблице
#### Отсортированная таблица по таблице ключей
Вывод отсортированной таблицы, используя для вывода отсортированную таблицу ключей
#### Отсортированная таблица
Вывод отсортированной таблицы
#### Сравнение методов сортировок
Вывод 2-х таблиц. 1-я содержит результаты сравнения двух методов сортировок при сортировке исходной таблицы. 2-я содержит результаты при использовании таблицы ключей.
#### Сравнение сортировки исходной таблицы и таблицы ключей
Вывод 2-х таблиц. 1-я содержит результаты сравнения сортировки исходной и таблицы ключей используя простой метод сортировки. 2-я содержит результаты при использовании ускоренного метода сортировки. 
### Описание реализуемой задачи
Реализовать следующие функции для работы со считанной таблицей
1) Добавление в конец таблицы нового элемента
2) Удаления элемента по цене
3) Вывод считанной таблицы
4) Поиск не новых машин указанной марки с один собствеником, отсутствием ремонтов, поддержкой обслуживания и в указанном диапазоне цен
5) Вывод таблицы ключей
6) Вывод отсортированной таблицы
7) Вывод отсортированной таблицы по таблице ключей
8) Сравнение эффективности по методу сортировки
9) Сравнение эффективности по сортируемой таблице
10) Завершение работы программы
### Способы обращения к программе
Запуск программы с помощью команды "./app.exe" через терминал. Далее будет предложено ввести номер команды. Для функций, работающих с таблицей, будет необходимо ввести путь к файлу. Для добавления необходимо ввести данные об элементе. Для удаления необходимо ввести марку и диапазон цен.

## Хранение структур в программе
Для хранения структур используется выделение динамической памяти. Начальный размер массива определяется количеством элементов при чтении данных из файла. При попытке добавить новый элемент в заполненный массив проводиться перевыделение памяти. Объем новой области памяти в 2 раза больше предыдущей.


```
Пока bottom <= top
	Для i от bottom до (top - 1)
		Если arr[i] > arr[i + 1]
			tmp = arr[i]
			arr[i] = arr[i + 1]
			arr[i + 1] = tmp
		Конец если
	Конец для
	
	top = top - 1
	
	Если bottom > top
		Завершить пока
	Конец если
	
	Для i от (top - 1) до bottom
		Если arr[i] > arr[i + 1]
			tmp = arr[i]
			arr[i] = arr[i + 1]
			arr[i + 1] = tmp
		Конец если
	Конец для

Конец пока
```

```С

pivot – опорный элемент
p – индекс опорного элемента после перестанови элементов


Функция quicksort(arr, low, high)
	Ecли low >= high
		Возврат
	
	pivot = arr[high]
	i = low - 1
	
	Для j от low до high - 1
		Если arr[i] < pivot
			i = i + 1

		Конец если
	Конец для

Конец функции
```

# Необходимые функции

### Чтение данных
`int read_car(FILE *f, car_t *car);` – читает один элемент из файла
- f – файловая перенная
- car – указатель, куда будет записаны считанные данные
`int read_cars(FILE *f, car_t **car_table, size_t *len);`
- f – файловая перенная
- cars – массив, куда будуд записаны считанные данные
- len – указатель на длину считаного массива
`int set_car_value(car_t *car, car_fields field, char *value);`
- car – указатель на элементы, в котором надо установить значение
- field – номер поля, которое будет заполняться
- value – значение, которое будет заполняться в поле

### Ввод данных 
`int input_car(car_t *car);` – ввод пользователем через потоковый ввод данных для одного элемента
`int input_filter(car_filter *filter);` – ввод пользователем через потоковый ввод данных для фильтра 

### Вывод данных
`void print_car(car_t *car, size_t int);` – вывод одного элемента
- car – указатель на элемент
- ind – порядковых индекс элемента
`void print_cars(car_t *car_table, size_t len);` – вывод таблицы элементов
- car_table – указатель на массив элементов
- len – длина таблицы
`void print_key_table_value(key_value_t *key);` – вывод одного элемента таблицы ключей
- key – указатель на элемент таблицы ключей
`void print_keys_table(key_value_t *key_table, size_t len);` – вывод таблицы ключей
- key_table – указатель на таблицу ключей
- len – длина таблицы ключей
`void print_cars_table_by_key_table(car_t *car_table, key_value_t *key_table, size_t len);` – вывод исходной таблицы в отсортированном виде, по таблице ключей
- car_table – таблица элементов
- key_table – отсортированная таблица ключей
- len – количество элементов в таблицах

### Сортировки
`int shaker_sort(void* pbeg, size_t count, size_t size, comparator_t cmp);` – универсальная сортировка шейкером
- pbeg – указатель на начало массива
- count – количество элементов
- size – размер одного элемента
- cmp – указатель на функцию сравнения элементов
`int quick_sort(void* pbeg, size_t count, size_t size, comparator_t cmp);` –универсальная быстрая сортировка
- pbeg – указатель на начало массива
- count – количество элементов
- size – размер одного элемента
- cmp – указатель на функцию сравнения элементов
`double cmp_car_prise(const void *a, const void *b);` – функция сравнения двух машин по цене, возвращает результат a - b
- a – элемент 1
- b – элемент 2
`double cmp_car_prise_by_key(const void *a, const void *b);` – функция сравнения двух ключей, возвращает результат a.prise - b.prise
- a – элемент 1
- b – элемент 2

### Функции таблицы
`int add_car(car_t **table, size_t *cur_size, size_t *max_size, car_t *new_car_t);` – добавление в конец таблицы нового элемента, если выделенная память закончилась, происходит перевыделение удвоенного размера
- table – таблица
- cur_size – указатель на текущий размер таблицы
- max_size – максимальный размер таблицы
`int remove_car(car_t *table, size_t *cur_size, double prise);` – удаление элемента по цене
- table – таблица
- cur_size – указатель на текущую длину массива
- prise – цена, элемент с которой надо удалить
`bool matches_filter(car_t *car, car_filter *filter);` – сравнение элемента с фильтром
- car – указатель на элемент
- filter – указатель на фильтр
`int get_table_key(car_t *table, key_value_t **key_table, size_t size);` – получение таблицы ключей, по исходной таблице
- table – таблица
- key_table – указатель на таблицу ключей
- size – размер таблиц
`void search_by_brand_prise(car_t *car_table, size_t len, car_t *table_filtered, size_t *len_filtered, car_filter *filter);` – поиск в таблице элементов по фильтру и заполнение ими новой таблицы
- car_table – таблица
- len –длина таблицы
- table_filtered – указатель на таблицу отфильтрованных элементов
- len_filtered – длина отфильтрованной таблицы
- filter – указатель на фильтр

# Оценка эффективности
## Вариативная часть
### Преимущества
1) Упрощенное представление в программе, так как все элементы являются одним типом данных и могут полностью храниться в одном массиве
2) Упрощенный доступ к данным, потому что все данных в памяти располагаются последовательно
### Недостатки
1) Сложность сортировки по полям, присущим не всем элементам
## Использование дополнительного массива ключей
### Преимущества
1) Упрощенная сортировка по массиву содержащему только одно поле от исходной структуры
### Недостатки
1) Дополнительные затраты на память
2) Увеличения объема кода, для реализации взаимодействия массива ключей и исходного массива

# Выводы
Использование вариативной части в структуре упрощает представление объектов в коде программы и делает код более компактным. Но при этом усложняются методы работы с таблицей при сортировке, удалении и добавлении, потому что появляется необходимость отслеживать какую вариативную часть имеет рассматриваемый объект. Для решения данной проблемы можно использовать таблицу ключей, которая содержит индекс элемента в исходной таблице и рассматриваемое значение. Но для данного решение необходимы дополнительные затраты по памяти.

По результатам сравнения можно сделать вывод, что при использовании таблицы ключей сортировке методом "шейкер" нужно на 74.54% меньше времени. Методу быстрой сортировки нужно на 70.07% меньше времени. 

При сравнении обычной и ускоренной для исходного массива и массива ключей можно сделать вывод, что при сортировке исходной матрицы при использование ускоренной сортировки уменьшает в среднем время работы на 89.14%, при сортировке таблицы ключей время уменьшается на 86.50%. Но для такого прироста, необходимы дополнительные затраты памяти для хранения таблицы ключей, в данном случает требуется на 16.67% больше памяти, чем для сортировки сортировки исходной матрицы

В результате, эффективность сортировки можно повысить дважды: при помощи ускоренного метода сортировки и использовании таблицы ключей. Что при работе с большими может давать значительное уменьшение времени работы алгоритма.
# Контрольные вопросы

### 1. Как выделяется память под вариантную часть записи?
Память под вариативную часть выделяться в виде динамического массива. Данный массив может содержать сами данные или указатели на элементы
### 2. Что будет, если в вариантную часть ввести данные, несоответствующие описанным?
Если в вариантивную часть ввести несоотствующие данные, это может привести к некоректной работе программы. Поэтому при вводе данных или их считывании их файла необходимо проводить проверку их соответствия описанным структурам
### 3. Кто должен следить за правильностью выполнения операций с вариантной частью записи?
За правильностью выполнения операций с вариативной частью должен следить программист. Необходимо внимательно отслеживать места выделения памяти, чтобы при завершении работы функции все полученные участки памяти были освобождены
### 4. Что представляет собой таблица ключей, зачем она нужна?
Таблица ключей в программе представляет собой массив данных, которые содержат информацию об индексе в исходной таблице и рассмастриваемое значения элемента с данным индексом. Данная таблица необходима для упрощения выполнения различных операция с большими таблицами. Таблица ключей помогает ускорить сортировку больших таблиц, так как необходимо перемещать по памяти объект меньшего размера.
### 5. В каких случаях эффективнее обрабатывать данные в самой таблице, а когда – использовать таблицу ключей?
Таблицу ключей эффективнее использовать в том случае, если работы с данными происходит по одному полю, например поиск по названию или сортировка по цене. Но при добавлении или удалении данных, поиске или сортировке по нескольким полям использование таблицы ключей может давать меньший прирост эффективности.
### 6. Какие способы сортировки предпочтительнее для обработки таблиц и почему?
Выбор сортировки зависит от размера данных и требований к затраченному времени сортировки и затрат памяти. Более медленные сортировки, такие как пузырек или вставками, может оказаться более предпочтительными при ограниченной памяти, так как не требуют дополнительных затрат, но время выполнения будет больше. Если же приоритетом является скорость выполнения, то в приоритете будут более быстрые сортировки, например быстрая или кучей, но из-за того, что они работают рекурсивно, то требуют больших затрат по памяти для стека вызовов, что особенно стоит учитывать при больших данных.
